Type of XSS

Reflected (non-persistent)

  • The most basic type

  • It is used immediately by server-side scripts to display page results without sanitizing the request.

  • Delivered via email or displayed on a neutral website.

  • The link causes victim's browsers to execute injected scripts.

Persistent (or stored)

  • Most devastating variant of XSS flaw.

  • Data made from attacker is displayed on web page and saved by the server.

  • When a user is browsing casually, their data is saved.

Server-side Versus DOM-based Vulnerablities

  • A server-side input of user is sent to the server then back to user as a web page.

  • In DOM-based, data does not touch server, it is reflected by Javascript code.

Self-XSS

  • Relies on social engineering to trick victims into executing malicious Javascript code into browser
    .

  • Not truly XSS vulnerability but poseses the same risks.

Mutated XSS (mXSS)

  • Attacker portrays something that is seemingly safe but rewritten by the browser.

  • Hard to detect with websites application login.
