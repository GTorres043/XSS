======= How Cross site Scripting works

The hacker will go into a web page and the has the vulnerability of cross site scripting being implemented into the web site. They then inspect/analyze the HTML used of the web site. After that, then the hacker will cut and paste the HTML into a text file, save it then open it in a web browser. HTTP is one of the reasons why cross site scripting is so successful- it accepts the request the hacker and builds a web page with the script embedded by the hacker. When the user enters their information, it is then stored on a web server. Once you click on the hyperlink in which the hacker has created an embedded the cross site scripting on all of your information directly goes to the hacker.
